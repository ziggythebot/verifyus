Status: Beta — Core verification system operational. Production deployment Q2 2026.

Architecture Overview

VerifyUS uses a three-layer verification architecture designed for ATS integration with minimal engineering overhead.

Core Components

  • Webhook Receiver: Intercepts application submissions from ATS platforms
  • Verification Engine: Multi-stage bot detection and identity verification
  • Result Injector: Returns verification status to ATS with metadata
┌─────────────────────────────────────────────────┐
│  APPLICATION FLOW                               │
├─────────────────────────────────────────────────┤
│                                                 │
│  ATS (Greenhouse/Lever/Workday)                │
│         ↓                                       │
│  VerifyUS Webhook                               │
│         ↓                                       │
│  Verification Engine                            │
│    ├─ L1: IP/Proxy Analysis                    │
│    ├─ L2: ID Verification + Liveness           │
│    └─ L3: Social Proof + Employment Check      │
│         ↓                                       │
│  Result → ATS (verified/rejected + metadata)   │
│                                                 │
└─────────────────────────────────────────────────┘

Integration Methods

Option 1: Manual Batch Processing (Recommended for Beta)

Export pre-screened applicants from your CRM/ATS and verify in batches. Only pay for candidates you've already filtered. No automatic webhook costs on every incoming application.

# Batch verification API
POST https://api.verifyus.com/v1/batch
Authorization: Bearer vk_live_YOUR_API_KEY

{
  "applicants": [
    {"id": "app_001", "email": "candidate1@example.com"},
    {"id": "app_002", "email": "candidate2@example.com"}
  ],
  "tier": "L2",
  "callback_url": "https://your-crm.com/webhooks/verifyus"
}

Option 2: Automatic Webhook Integration

Real-time verification for every incoming application. Best for high-volume roles with severe bot spam.

# Configure webhook in your ATS
POST https://api.verifyus.com/v1/webhooks
Authorization: Bearer vk_live_YOUR_API_KEY

{
  "ats_provider": "greenhouse",
  "target_url": "https://boards-api.greenhouse.io/v1/boards/YOUR_BOARD/jobs",
  "verification_tier": "L2",
  "auto_reject": true
}

Verification Tiers

L1: Pre-Filter ($1/app)

  • IP address analysis (VPN/proxy detection)
  • Device fingerprinting
  • Rate limiting (application velocity)
  • Blocks: 40-60% of bot traffic

L2: Standard ($3/app) [RECOMMENDED]

  • All L1 checks
  • Government ID verification (Passport, Driver's License, State ID)
  • Biometric liveness detection (webcam-based, no app required)
  • Work authorization check (US-only, expandable)
  • Blocks: 95%+ of bot traffic

L3: Deep Scan ($4/app)

  • All L2 checks
  • LLM-powered social proof scraping (LinkedIn, GitHub, Twitter)
  • Employment history cross-check
  • Dedicated Slack channel for high-risk alerts
  • Blocks: 99.9% of bot traffic

Security & Compliance

Zero-Knowledge Proof Architecture

VerifyUS uses zkPass cryptographic proofs anchored on Ethereum for tamper-proof identity verification. AI-powered verification engine + blockchain attestation.

┌─────────────────────────────────────────────────┐
│  VERIFICATION FLOW (zkPass + Ethereum)          │
├─────────────────────────────────────────────────┤
│                                                 │
│  Applicant uploads ID                           │
│         ↓                                       │
│  AI verification layer (liveness + ID check)    │
│         ↓                                       │
│  Generate zkProof (no PII stored)               │
│         ↓                                       │
│  Anchor proof hash to Ethereum                  │
│         ↓                                       │
│  Return verification token to recruiter         │
│                                                 │
│  ✓ Recruiter sees: VERIFIED                    │
│  ✓ Blockchain record: proof hash only           │
│  ✓ VerifyUS storage: ZERO identity data         │
│                                                 │
└─────────────────────────────────────────────────┘

What We NEVER Store

  • Passport/ID document scans
  • Social Security Numbers or tax IDs
  • Biometric templates (facial recognition data)
  • Government ID numbers

We only store: Cryptographic proof hashes anchored to Ethereum. The blockchain becomes the immutable verification ledger — no central honeypot of identity data.

Compliance & Regulatory Framework

VerifyUS is designed for cross-border hiring compliance — US, EU, UK regulations met by default using anonymized cryptographic proofs instead of centralized PII storage.

Regulation Requirement VerifyUS Solution
GDPR (EU) Right to erasure, data minimization Zero PII storage — only proof hashes on Ethereum
CCPA (California) Consumer data rights, opt-out No personal data retained = no breach risk
UK DPA 2018 Lawful basis for processing Cryptographic verification without storing biometrics
SOC 2 Type II Security controls audit Certification Q3 2026

Data Sovereignty

  • Verification happens locally: AI processing in-region (US-East, EU-West)
  • Blockchain anchoring: Public Ethereum mainnet (global) or Polygon (cost-optimized)
  • No data export: Identity documents never leave verification node
  • Anonymized proofs: Blockchain stores cryptographic hashes, not identity attributes

Technical Security

  • Encryption: TLS 1.3 for all API traffic, AES-256 at rest
  • Access control: Zero-trust architecture, API keys rotated every 90 days
  • Audit trail: Immutable verification log on Ethereum (query by proof hash)

Swift Implementation

import VerifyUS

// Initialize SDK
let verifyUS = VerifyUSClient(apiKey: "vk_live_YOUR_API_KEY")

// Trigger verification flow
verifyUS.verify(
    applicantID: "app_12345",
    tier: .standard,
    presentingViewController: self
) { result in
    switch result {
    case .success(let verification):
        print("Verified: \(verification.verificationID)")
        print("Confidence: \(verification.confidenceScore)")

    case .failure(let error):
        print("Verification failed: \(error.localizedDescription)")
    }
}

Objective-C Implementation

#import <VerifyUS/VerifyUS.h>

// Initialize SDK
VUSClient *client = [[VUSClient alloc] initWithAPIKey:@"vk_live_YOUR_API_KEY"];

// Trigger verification flow
[client verifyWithApplicantID:@"app_12345"
                         tier:VUSTierStandard
        presentingViewController:self
                   completion:^(VUSVerification *verification, NSError *error) {
    if (verification) {
        NSLog(@"Verified: %@", verification.verificationID);
    } else {
        NSLog(@"Error: %@", error.localizedDescription);
    }
}];
